# explain 中的 join type

#

# type

type 表示 join type ，有如下值：

# system

表示该表只有一行记录。

# const

表示使用索引一次就得到了该表。通常在主键索引或唯一索引和常数比较的时候出现。

示例：

explain
select author_id
from article where id = 3
;

# eq_ref

# possible_keys

possible_keys 表示在得到该表时可能会用到的索引列。

如果该字段的值为 null ，则表示在得到该表时可能不会使用索引，也即存在优化的空间。

# key

key 表示在得到该表时实际用到的索引列。

这里面可能存在一种情况：key 里面出现的索引列在 possible_keys 字段中没有出现。

如果该字段的值为 null ，则表示在得到该表时可能不会使用索引，也即存在优化的空间。

# key_len

key_len 表示 key 字段里索引的长度（字节数）。是最大可能长度，并非实际使用长度。越短越好。

# ref

ref 通常和 key 一起使用。展示的是 key 字段里的索引列的比较对象。可能是一个 const ，也可能是其他表中的某一列。

示例：

explain
select *
from dept_emp de
left join employees e on de.emp_no = e.emp_no
where de.dept_no = 'd005'
;

从 explain 语句的结果可以知道，先查 de 表，此时 de 表的索引 dept_no 和常数 d001 比较，所以 ref 字段显示为 const 。

然后查 e 表，e 表的索引是主键 emp_no ，和 de 表的 emp_no 字段对比，所以 ref 字段显示为 employees.de.emp_no 。

# rows

rows 表示 MySQL 执行查询时需要扫描的行数，是一个估算值，不是一个精确值。
