# TCP 粘包拆包

# 什么是 TCP 粘包拆包

# MTU

MTU 指数据链路层的最大传输单元，也即最大的数据包的大小，单位是字节。我们普遍使用的以太网 MTU 值是 1500 字节。可以通过下面的命令查看：

cat /sys/class/net/eth0/mtu

# MSS

MSS 指 TCP 连接的客户端和服务端约定的最大的 TCP 报文长度。如果 MTU 的值是 1500 ，那么 MSS 的值就是：

1500 - 20(IP Header) - 20(TCP Header) = 1460 Byte

# 什么情况下会发生粘包

  • 客户端写入的数据小于 socket 缓冲区的大小，会将多次写入的数据合并成一次发送，发生粘包。
  • 服务端不及时读取 socket 缓冲区里的数据，即客户端多次发送的数据被服务端一次读取，发生粘包。

# 什么情况下会发生拆包

  • 客户端写入的数据大于 socket 缓冲区的大小，会将本次写入的数据拆成多次发送，发生拆包。
  • 当 TCP 报文长度 - TCP 头部长度 > MSS 时，会发生拆包。

# 如何解决 TCP 粘包拆包问题

四种方案：

  • 固定消息的长度。
  • 消息里使用分隔符。
  • 消息里自带长度。
  • 自定义协议和编解码器。
