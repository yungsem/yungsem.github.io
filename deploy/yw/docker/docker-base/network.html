# 网络模式

# 四种网络模式

  • bridge
  • host
  • container
  • none

# bridge

当我们启动 docker 时，会在主机上创建了一个名为 docker0 的虚拟网桥，虚拟网桥类似于物理交换机。如下图所示：

当我们启动一个新容器时，系统从 docker0 的子网中分配一个 IP 给该容器，并将容器的默认网关设置为 docker0 的 IP 地址。

然后在主机上创建一对虚拟网卡 veth pair ，将 veth pair 的一端放在新创建的容器中，命名为 eth0 ，这个 eth0 就作为容器的网卡，另一端放在主机中，以 vethXXX 命名，如下图所示：

创建好的 vethXXX 设备会被加入 docker0 网桥中。可以通过下面的命令查看：

brctl show
bridge name	bridge id		STP enabled	interfaces
docker0		8000.0242a51697f6	no		veth11e0d60
							            vethc57bfe0

完整的网络原理示意图如下：

借助虚拟网桥，运行在该主机上容器与容器之间，容器与主机之间都可以进行网络通信。

bridge 模式是 docker 的默认网络模式，如果我们在执行 docker run 命令启动容器时，不使用 --network 参数指定网络模式，就使用默认的 bridge 模式。

# host

如果在启动一个容器时指定网络模式为 host ，系统就不会为该容器创建单独的网络空间，而是直接使用主机的网络空间，即使用主机的 IP 地址和端口。也不会用到 docker0 网桥。

# container

如果在启动容器 A 时指定网络模式为另外一个容器 B （命令为：--network container:containerName），则系统不会为容器 A 创建单独的网络空间，而是让其共用容器 B 的网络空间。

# none

使用 none 模式，容器有自己的网络空间，但是没有配置网络，没有网卡，IP 和端口等。
