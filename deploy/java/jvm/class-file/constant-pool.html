# 常量池

# 对常量池的理解

一个类有一系列的“名称”，例如：

  • 类名
  • 字段名
  • 方法名
  • 父类名
  • 实现的接口名
  • 等等

这些名称在类文件中是集中管理的，就放在常量池中。

每一种名称都有自己的结构，如下表所示：

# CONSTANT_Utf8_info

表示：字符串。

结构：

CONSTANT_Utf8_info {
    u1 tag;
    u2 length;
    u2 btyes[length];
}
  • tag 是 CONSTANT_Utf8 ，值是 1 。
  • length 是该字符串对应的字节数组的长度。
  • bytes[] 是该字符串对应的字节数组。

# CONSTANT_Class_info

表示：类或接口全名。

结构：

CONSTANT_Class_info {
    u1 tag;
    u2 name_index;
}
  • tag 是 CONSTANT_Class ，值是 7 。
  • name_index 是类或接口的全名（CONSTANT_Utf8_info）在常量池中的索引值。

# CONSTANT_NameAndType_info

表示：字段或方法的 name + descriptor 。

结构：

CONSTANT_NameAndType_info {
    u1 tag;
    u2 name_index;
    u2 descriptor_index;
}
  • tag 是 CONSTANT_NameAndType ，值是 12 。
  • name_index 是该字段或方法的 name （CONSTANT_Utf8_info）在常量池中的索引值。
  • descriptor_index 是该字段或方法的 descriptor （CONSTANT_Utf8_info）在常量池中的索引值。

# CONSTANT_Fieldref_info

表示：字段。

结构：

CONSTANT_Fieldref_info {
    u1 tag;
    u2 class_index;
    u2 name_and_type index;
}
  • tag 是 CONSTANT_Fieldref ，值是 9 。
  • class_index 该字段所在的类或接口（CONSTANT_Class_info）在常量池中的索引值。
  • name_and_type index 是该字段的 name + descriptor （CONSTANT_NameAndType_info） 在常量池中的索引值。

# CONSTANT_Methodref_info

表示：类的方法

结构：

CONSTANT_Methodref_info {
    u1 tag;
    u2 class_index;
    u2 name_and_type index;
}
  • tag 就是 CONSTANT_Methodref ，值是 10 。
  • class_index 是该方法所在的类（没有接口）（CONSTANT_Class_info）在常量池中的索引值。
  • name_and_type index 是该方法的 name + descriptor （CONSTANT_NameAndType_info） 在常量池中的索引值。

# CONSTANT_InterfaceMethodref_info

表示：接口的方法

结构：

CONSTANT_InterfaceMethodref_info {
    u1 tag;
    u2 class_index;
    u2 name_and_type index;
}
  • tag 就是 CONSTANT_InterfaceMethodref ，值是 11 。
  • class_index 是该方法所在的接口（没有类）（CONSTANT_Class_info）在常量池中的索引值。
  • name_and_type index 是该方法的 name + descriptor （CONSTANT_NameAndType_info） 在常量池中的索引值。

# CONSTANT_Methodtype_info

表示：方法的 descriptor （类型）

结构：

CONSTANT_Methodtype_info {
    u1 tag;
    u2 descriptor_index;
}
  • tag 就是 CONSTANT_Methodtype ，值是 16 。
  • descripter_index 是该方法的 descriptor （CONSTANT_Utf8_info） 在常量池中的索引值。

# CONSTANT_String_info

表示：一个 Strng 类型的实例

结构：

CONSTANT_String_info {
    u1 tag;
    u2 string_index;
}
  • tag 是 CONSTANT_String ，值是 8 。
  • string_index 是该字符串对应（CONSTANT_Utf8_info）在常量池中的索引值。

# CONSTANT_Integer_info

表示：整型常量

结构：

CONSTANT_Integer_info {
    u1 tag;
    u4 bytes;
}
  • tag 是 CONSTANT_Integer ，值是 3 。
  • bytes 是这个整型常量对应的二进制值。

# CONSTANT_Long_info

表示：长整型常量。一个 CONSTANT_Long_info 占两个 constant_pool entry 的长度。

结构：

CONSTANT_Long_info {
    u1 tag;
    u4 high_bytes;
    u4 low_bytes;    
}
  • tag 是 CONSTANT_Long ，值是 5 。
  • hight_bytes 和 low_bytes 共同表示的是这个长整型常量对应的二进制值。

# CONSTANT_Float_info

略

# CONSTANT_Double_info

略

# CONSTANT_MethodHandle_info

表示：method handle

结构：

CONSTANT_MethodHandle_info {
    u1 tag;
    u2 reference_kind;
    u2 reference_index;
}

  • tag 是 CONSTANT_MethodHandle ，值是 15 。

  • reference_kind 的值在 [1,9] 之间，表示的是该 method handle 的类型。

  • reference_index

    如果 reference_kind 的值是下列值之一：

    • 1 (REF_getField)
    • 2 (REF_getStatic)
    • 3 (REF_putField)
    • 4 (REF_putStatic)

    那么，reference_index 表示的是该 method handle 对应的 field （CONSTANT_Fieldref_info）在常量池中的索引值。

    如果 reference_kind 的值是下列值之一：

    • 5 (REF_invokeVirtual)
    • 8 (REF_newInvokeSpecial)

    那么，reference_index 表示的是该 method handle 对应的类的 method 或 constructor （CONSTANT_Methodref_info）在常量池中的索引值。

    如果 reference_kind 的值是下列值之一：

    • 6 (REF_invokeStatic)
    • 7 (REF_invokeSpecial)

    那么，reference_index 表示的是该 method handle 对应的类或接口的 method （CONSTANT_InterfaceMethodref_info）在常量池中的索引值。

    如果 reference_kind 的值是下列值之一：

    • 9 (REF_invokeInterface)
    • 7 (REF_invokeSpecial)

    那么，reference_index 表示的是该 method handle 对应的接口的 method （CONSTANT_InterfaceMethodref_info）在常量池中的索引值。

# CONSTANT_Dynamic_info

表示：

结构：

CONSTANT_Dynamic_info {
    u1 tag;
    u2 bootstrap_method_attr_index;
    u2 name_and_type_index
}
  • tag 是 CONSTANT_Dynamic ，值是 17 。
  • bootstrap_method_attr_index The value of the bootstrap_method_attr_index item must be a valid index into the bootstrap_methods array of the bootstrap method table (§4.7.23) of this class file.
  • name_and_type_index 是该方法的 name + type （CONSTANT_NameAndType_info） 在常量池中的索引。

# CONSTANT_InvokeDynamic_info

CONSTANT_InvokeDynamic_info 被 invokedynamic 命令使用， 作用如下：

  • specify a bootstap method
  • specify the dynamic invocation name
  • specify the argument and return types of the call
  • specify a sequence of additional constants called static arguments to the bootstap method

结构：

CONSTANT_InvokeDynamic_info {
    u1 tag;
    u2 bootstrap_method_attr_index;
    u2 name_and_type_index
}
  • tag 是 CONSTANT_InvokeDynamic ，值是 18 。
  • bootstrap_method_attr_index The value of the bootstrap_method_attr_index item must be a valid index into the bootstrap_methods array of the bootstrap method table (§4.7.23) of this class file.
  • name_and_type_index 是该方法的 name + type （CONSTANT_NameAndType_info） 在常量池中的索引。

# CONSTANT_Module_info

表示：一个 module 。

结构：

CONSTANT_Module_info {
    u1 tag;
    u2 name_index;
}
  • tag 是 CONSTANT_Module ，值是 19 。
  • name_index 是该 module 的名称（CONSTANT_Utf8_info）在常量池中的索引值。

只有在类文件的 access_flags 含有 ACC_MODULE flag 时，该类文件的常量池中才能包含 CONSTANT_Module_info ，否则不能包含。

# CONSTANT_Package_info

表示：一个 package 。

结构：

CONSTANT_Package_info {
    u1 tag;
    u2 name_index;
}
  • tag 是 CONSTANT_Package ，值是 20 。
  • name_index 是该 package 的名称（CONSTANT_Utf8_info）在常量池中的索引值。

只有在类文件的 access_flags 含有 ACC_MODULE flag 时，该类文件的常量池中才能包含 CONSTANT_Package_info ，否则不能包含。
