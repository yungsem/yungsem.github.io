# 原型链

# 写在前面

JS 没有类的概念，只有对象（或者称为实例），所以就不存在根据类来创建实例这一说，只存在根据实例创建实例，或者说成 根据父实例创建出子实例 。

# Function

一切都从 Function 开始。Function 是一个一级实例。

通过 Function ，我们可以创建出 Function 这个一级实例的子实例（下面称为 Function 的子实例）。

我们可以通过下面两种方式来创建 Function 的子实例：

// 方式一
var sum1 = new Function('a', 'b', 'return a + b');

// 方式二
function sum2(a, b) {
    return a + b
}

console.log(typeof sum1); // function
console.log(typeof sum2); // function

可以发现，所谓的 创建 Function 的子实例 其实就是定义函数。

没错，我们每定义一个函数，其实就是创建了一个 Function 的子实例。

在上面的例子中，sum1 和 sum2 都是 Function 的子实例。

但是，有一点我们要格外注意：上面我们说 Function 是一个一级实例，那这个一级实例有没有父实例呢？答案是有，并且它的父实例就是它自身。所以 Function 不仅仅是一个一级实例，同时也是一个顶级实例。

下面的输出验证了这一点，Function 也是通过 function 关键字定义出来的函数，既然是定义出来的一个函数，那么它就是 Function 的一个子实例。

console.log(Function.toString()) // function Function() { [native code] }

JS 内置了不少标准对象，如：

  • Object
  • String
  • Array
  • ...

这些标准对象其实也是通过 function 关键字定义出来的函数：

console.log(Object.toString()) // function Object() { [native code] }
console.log(String.toString()) // function String() { [native code] }
console.log(Array.toString()) // function Array() { [native code] }

所以，像 Object, String, Array 等等这些标准对象都是 Function 的子实例。

通常情况下，我们又把这些标准对象称为构造函数，因为我们可以通过这些标准对象 new 出来相应的子对象（子实例）：

var obj = new Object();
var str = new String();
var arr = new Array();

上面创建出来的 obj 是 Object 的子实例，str 是 String 的子实例，arr 是 Array 的子实例。

到此为止，我们实际上已经引出了两个层级的子实例：

  • Object, String, Array 是 Function 的子实例
  • obj, str, arr 分别是 Object, String, Array 的子实例

其实还有一个层级的子实例，就是 Function 是 Function 的子实例。

除了 JS 内置的构造函数（就是内置的标准对象），我们也可以定义自己的构造函数：

function Person(name, age) {
    this.name = name;
    this.age = age;
}

var person = new Person('ys', 25);

还是同样的道理，Person 是 Function 的子实例，person 是 Person 的子实例。

Person 其实和 Object, String, Array 这些内置的标准对象没什么不同，只不过前者是我们自己定义的，后者是 JS 标准定义的。但是它们处于同一层级，不要区别看待。

到这里，我们发现，JS 里的实例是有层级关系的，JS 正是通过实例的这种层级关系实现了继承。总结如下：

  • 顶级实例：Function
  • 一级实例（顶级 Function 的子实例）：Function
  • 二级实例（Function 的子实例）：Object, Array, String, Person （所有的函数）
  • 三级实例（Object, Array 等的子实例）： obj, arr （借助构造函数 new 出来的实例）

# prototype 和 __proto__

在上面列出的实例的层级体系中，具有父实例身份的实例，都有一个 prototype 属性。也就是说，三级实例没有 prototype 属性，其他层级的实例都有 prototype 属性。

这个 prototype 属性到底有什么用？继承。子实例会继承父实例 prototype 属性里定义的属性和方法。prototype 属性其实也是一个实例，也有层级关系，下面会讲。

具有子实例身份的实例，都有一个 __proto__ 属性。

那么 __proto__ 又是干什么的呢？子实例的 __proto__ 属性等于父实例的 prototype 属性。

# 原型链（由下向上 __proto__）

理清了实例的层级关系，也搞清了 prototype 和 __proto__ ，原型链就出来了。

Function 是定级实例，具有父实例的身份，所以它有一个 prototype 属性。

但 Function 同时又是一级实例，是顶级实例的子实例，所以它还有一个 __proto__ 属性。

根据 子实例的 __proto__ 属性等于父实例的 prototype 属性。 ，我们可以得出：

Function.__proto__ === Function.prototype

原因上面已经说明过了，Function 的父实例还是 Function 。

验证一下：

console.log(Function.__proto__ === Function.prototype); // true

再往下，我们看二级实例，它们是一级 Function 的子实例，如 Object, Array 等。

这些二级实例一方面是一级 Function 的子实例，另一方面又是 obj, arr 等三级实例的父实例，所以它们也是既有 __proto__ 属性，也有 prototype 属性。

它们的 __proto__ 属性等于父实例（一级 Function）的 prototype 属性。

console.log(Object.__proto__ === Function.prototype); // true
console.log(String.__proto__ === Function.prototype); // true
console.log(Array.__proto__ === Function.prototype); // true

再看三级实例，三级实例不具有父实例的身份，所以没有 prototype 属性，但是有子实例的身份，所以具有 __proto__ 属性。它们的 __proto__ 属性等于父实例（Object, Array, Person）的 prototype 属性。

var person = new Person('ys', 25);
var obj1 = new Object();
var obj2 = {};
var str1 = new String();
var str2 = 'abc';
var arr1 = new Array();
var arr2 = [];

console.log(obj1.__proto__ === Object.prototype); // true
console.log(obj2.__proto__ === Object.prototype); // true
console.log(str1.__proto__ === String.prototype); // true
console.log(str2.__proto__ === String.prototype); // true
console.log(arr1.__proto__ === Array.prototype); // true
console.log(arr2.__proto__ === Array.prototype); // true
console.log(person.__proto__ === Person.prototype); // true

至此，已经形成一条右下向上的继承链，总结如下：

obj.__proto__ === Object.prototype
Object.__proto__ === Function.prototype
Function.__proto__ === Function.prototype

# 原型链（Prototype）

在上一小节，我们梳理了由下向上的继承链，其实就是原型链。在这个原型链中，Function.prototype 仿佛位于顶端。实际不然。

我们上面只研究了各级实例的 __proto__属性，我们还需要研究一下各级实例的 prototype 属性。

这次我们从上向下看，首先是 Function.prototype 。Function.prototype 其实是 Object 的子实例，验证一下：

console.log(Function.prototype.__proto__ === Object.prototype); // true

接下来是二级实例的 prototype 属性：

console.log(Array.prototype.__proto__ === Object.prototype); // true
console.log(String.prototype.__proto__ === Object.prototype); // true
console.log(Person.prototype.__proto__ === Object.prototype); // true

可以发现，二级实例的 prototype 也都是 Object 的子实例。

但是 Object.prototype 不是 Object 的子实例。

console.log(Object.prototype.__proto__ === Object.prototype); // false

那么 Object.prototype 是什么呢？其实，Object.prototype 可以看作是 null 的子实例，因为：

console.log(Object.prototype.__proto__ === null); // true

我们再来总结一下： 一级实例和二级实例（研究 prototype 属性时，顶级 Function 和 一级 Function 看作一个）都有 prototype 属性，三级实例没有。

一级实例和二级实例（Object 除外）的 prototype 都是 Object 的子实例。

Object.prototype 是 null 的子实例。

至此，原型链已经全部出来了，位于原型链顶端的是 null 。

最后，理解 JS ，我们应该抛弃类和对象的概念，JS 中没有类，并不是像 Java 那样，根据类创建对象。在 JS 中只有对象（或者称为实例），全部都是实例。我们可以根据父实例创建子实例，同时，利用原型链的机制，子类会继承父实例的属性和方法。
