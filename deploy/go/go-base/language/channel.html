# channel

# 简述

channel 是一个通道，goroutine 可以向 channel 里写入数据，也可以从 channel 里读取数据。通常情况下，往 channel 里写入数据和从 channel 里读取数据的是不同的 goroutine 。所以 channel 又被形象的比喻为胶水，将 goroutine 胶合在一起。如下图所示：

# 操作 channel

# 创建

// 创建一个无缓冲的 channel
ch := make(chan int)

// 创建一个有缓冲的 channel
ch := make(chan int, 5)

// 创建一个只读的 channel
ch := make(<-chan int)

// 创建一个只写的 channel 
ch := make(chan<- int)

# 写入

ch := make(chan int)

// 往 ch 中写入数值 5
ch<- 5

往一个已经满了的 channel 中写入数据，执行写入操作的 goroutine 会阻塞。

# 读取

ch := make(chan int)

// 从 ch 中读取一个值，并赋值给变量 i
i := <-ch

从一个空的 channel 中读取数据，执行读取操作的 goroutine 会阻塞。

# 关闭

我们可以使用内置的 close 函数关闭一个 channel ：

ch := make(chan int)

// 关闭 ch
close(ch)

一般由上游的 goroutine 执行关闭 channel 的操作，下游的 goroutine 可以直接从被关闭的 channel 中读取数据而不会阻塞，可以无限次读取。而且每次可以读出两个数据，一个是 channel 里元素类型的零值，一个是 ok ，当 ok = false ，表示 channel 已经被关闭。

这个特性对下游的 goroutine 比较重要，下游的 goroutine 可以根据 channel 是否被关闭执行对应的逻辑，而不是一直傻等上游 goroutine 往 channel 里写入数据。

ch := make(chan int)

close(ch)

// i = 0
// ok = false
i, ok := <- ch

一个 channel 可以被多个下游 goroutine 同时读取数据，当上游 goroutine 关闭 channel 时，所有阻塞在这个 channel 上的下游 goroutine 都会同时感知到 channel 被关闭了。这实际上可以用于 等待-通知 的场景里，上游 goroutine 关闭 channel ，所有下游 goroutine 都被通知到，然后执行各自对应的逻辑。

func main() {
	ch := make(chan int)

	var wg sync.WaitGroup
	wg.Add(5)

	for i := 0; i < 5; i++ {
		go func(n int) {
			defer wg.Done()
			
			<-ch
			fmt.Printf("down goroutine %d informed\n", n)
		}(i)
	}

	fmt.Printf("upper goroutine close channel\n")
	close(ch)

	wg.Wait()
}

// output
upper goroutine close channel
down goroutine 1 informed
down goroutine 4 informed
down goroutine 0 informed
down goroutine 2 informed
down goroutine 3 informed

# 遍历

遍历一个 channel 也是给下游 goroutine 提供的一个操作。下游 goroutine 可以使用 for ... range 来遍历一个 channel 。for ... range 会自动判断 channel 是否已经被关闭来终止循环。

在下面的例子中，上游 goroutine 循环往 channel 写入 5 个数字，然后关闭 channel 。下游 goroutine 使用 for ... range 遍历这个 channel ，并且顺利终止循环：

func main() {
	ch := make(chan int)

	go func() {
		defer close(ch)
		for i := 0; i < 5; i++ {
			ch<- i
		}
	}()

	for v := range ch {
		fmt.Printf("%d ", v)
	}
}

// output
0 1 2 3 4

# channel 所有权

未被初始化的 channel 是一个 nil channel 。如下：

// ch is nil
var ch chan int

对 nil channel 进行读、写或关闭操作，都会触发 panic ：

// panic
ch<- 5
// panic
<- ch
// panic
close(ch)

向一个已经 close 的channel 中写入数据，也会触发 panic ：

ch := make(chan int)
close(ch)
// panic
ch<- 5

为了规避上述问题，我们在使用 channel 时必须明确 channel 的所有权。即哪个 goroutine 是该 channel 的主人。

channel 的主人应该负责：

  • 初始化该 channel
  • 执行写入操作
  • 关闭该 channel
  • 转让所有权给其他 goroutine

channel 的主人可以将上述职责封装起来，然后暴露出一个只读的 channel 给下游 goroutine 使用。
