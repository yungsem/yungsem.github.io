# BFC 和 IFC

# 概述

我们都知道，html 文档中的所有元素最终会共同形成一棵 DOM 树。而每一个 html 元素都对应一个盒子，这些盒子共同组织在一起，又形成了 html 文档的盒子体系。盒子体系不是树形结构，而是嵌套结构。

我们下面要讲的 BFC 和 IFC 最根本的作用就是维系这个盒子体系的平衡。

# BFC

# 哪些元素会生成 BFC

BFC 是由元素生成的，那么哪些元素会生成 BFC 呢？如下：

  • html 元素（根元素）
  • display 值为 inline-block 或者 table-cell 或者 table-caption 的元素
  • overflow 值为 hidden 或者 auto 或者 scroll 的元素
  • float 值不为 none 的元素
  • position 值不为 static 和 relative 的元素

其中，html 元素生成的 BFC 是应用最广泛的 BFC ，我们可以称之为根 BFC 。html 元素的所有子孙元素，只要不满足生成独立 BFC 的条件，就都会参与到这个根 BFC 中来。

上面说到，盒子体系是嵌套结构，而 BFC 的作用就是约束这些盒子，那么 BFC 到底是通过什么来约束多层级嵌套的盒子的呢？答案是 containing block 。

# BFC 对盒子的约束

BFC 只约束 block-level box 。处在同一个 BFC 里所有的不同层级的 block-level box 都必须遵守如下约束：

  • 每个 block-level box 都在自己的 containing block 里平铺占满一行（独占一行）
  • 同一个 containing block 里的多个 block-level box 在这个 containing block 里从上到下依次排列（纵向排列）
  • 同一个 containing block 里相邻的两个 block-level box 会发生 margin 重叠（margin 重叠）

BFC 自身还具有以下三个特性：

  • 独立性：不和同一级的其他 BFC 重叠。BFC 也可以嵌套，一旦嵌套可以理解为发生了重叠，但这种重叠并不是发生在同一级
  • 封闭性：BFC 里面翻江倒海，不影响外面。BFC 就是一个封闭的空间，外面不知道里面发生着什么
  • 兼容性：BFC 兼容 IFC 。inline-level box 也可以在 BFC 中显示，但是显示之前必须先在 BFC 的基础上生成一个独立的 IFC ，然后再参与这个 IFC ，接受这个 IFC 的约束

# IFC

# 哪些元素会生成 IFC

IFC 也是由元素生成的，那么哪些元素会生成 IFC 呢？如下：

  • display 值为 inline 的元素
  • 如果一个元素生成的盒子里放的全都是 inline-level box ，那么这个元素会先生成 IFC

# IFC 对盒子的约束

IFC 只约束 inline-level box 。IFC 都是临时生成的，同一个 IFC 不会贯穿多个层级，这一点和 BFC 不同。所以处在同一个 IFC 里的 inline-level box 都是同级的，它们必须遵守如下约束：

  • 每个 inline-level box 都从 containing block 的 top edge 和 left edge 开始，从左到右一个接着一个排列，一行会有多个 inline-level box （横向排列）
  • 水平方向上相邻的两个 inline-level box 不会发生 margin 重叠，各自的 border 和 padding 也不会重叠（无 margin 重叠）
  • 排在一行的 inline-level box 形成一个虚拟的长方形，这个长方形称为 line box （生成 line box）
  • 如果某个 inline-level box 的宽度超过了一行，则自动换行，但是如果这个 inline-level box 无法换行，则溢出 line box 。如果一个 inline-level box 自动换行了，这个 inline-level box 的 margin, border, padding 在换行处不会生效 （自动换行）

# 总结

BFC 和 IFC 从本质上来讲就是一组约束规则。BFC 专门约束 block-level box ，IFC 专门约束 inline-level box 。

两者之中，BFC 尤为重要，是 html 盒子体系的基石。
