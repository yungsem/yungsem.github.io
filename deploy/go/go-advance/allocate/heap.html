# 堆内存

# 什么是 Go 的堆内存

每个进程都有一块自己独享的虚拟内存。

一个 Go 程序运行起来的时候，对应一个 Go 进程，该 Go 进程有一块独享的虚拟内存。

我们所说的堆内存，就是指 Go 进程独享的这一块虚拟内存。

# Go 的堆内存的分级

Go 对堆内存进行分级管理，如下：

内存类型 说明
page 大小 8 KB
mspan 由若干个连续的 page 组成，真正分配对象的最小内存单位
mcache 每个逻辑处理器（P）都有一个 mcache
mcentral mcentral 中有 mspan 供 mcache 申请领用
mheap 就是堆内存本身，通过 mcentral 达到分级管理内存的目标

# page

一个 page 的大小是 8 KB 。

# mspan

一个 mspan 由多个连续的 page 组成。

mspan 根据大小的不同，分为 67 类。类型的区分只是针对大小，所有类型的 mspan 结构都是一致的，是一个双向链表。

# mcache

一个 mcache 就是一个数组，长度为 134 ，数组元素就是 mspan 。

也就是说，一个 mcache 就是一个包含着 134 个 mspan 的数组。

# mcentral

mcentral 就是 mspan 的中心。上面提到，mspan 按大小分为 67 类，每种类型的 mspan 都有一个对应的 mcental 。

每个 mcentral 持有着两个 mspan_list ，分别为 nonempty_list 和 empty_list 。

nonempty_list 里的 mspan 一定有空闲空间。

empty_list 里的 mspan 不确定有没有空闲空间。

# mheap

mheap 就是 Go 进程拥有的那一块虚拟内存，也即堆内存。

mheap 持有 134 个 mcentral 。
