# QoS

# 简述

QoS 是 Quality of Service 的缩写，意为服务质量。是 MQTT 协议规范里的概念。

QoS 一共有三个等级，分别为：

  • level 0
  • level 1
  • level 2

MQTT 根据上述三个等级传递消息。注意这里所说的传递消息，指的是发送方将消息发送到 EMQX 服务器。

# QoS level 0

QoS = 0 ，消息最多只发送一次。

发送方依赖底层 TCP/IP 协议将消息发送到 EMQX 服务器，服务器不返回确认，发送失败了发送方也不会重试。

这样就导致 EMQX 服务器可能接收到消息，也可能没接收到消息。

# QoS level 1

QoS = 1 ，消息至少发送一次。

服务器收到消息后会返回确认，一个 PUBBACK 消息。下列三种情况会导致发送方重新发送消息：

  • 发送方连接失败导致消息发送失败
  • 发送放本身故障导致消息发送失败
  • 发送方在规定时间内未收到服务器返回的确认消息

重新发送的消息的头部会有一个 DUP 信息位。

SUBCRIBE 和 UNSUBCRIBE 消息都使用 QoS = 1 。

# QoS level 1

QoS = 2 ，唯一传递（不重复传递）。

服务器一定会收到发送方发来的消息，且相同的消息只会收到一次。
