# 通配符

# 通配符的引入

我们都知道，List<E> 是一个 generic class ，我们在使用它的时候要将其具体化，比如 List<Integer>，List<Double> 等等。

但是有时候我们在具体化的时候，并不是很明确我要用一个什么类型的 type argument 来具体化它。也有的时候，为了使代码更加灵活，我们并不想将其具体化为某一个类型，而是想将其具体化为某一组类型。

例如，我们现在有下面这个方法，它接收 List<Integer> ，返回该 List 中所有元素的和。

public static double sumOfIntList(List<Integer> list){
    // code
}

我们还有一个类似的方法，它接收 List<Double> ，也返回该 List 中所有元素的和。

public static double sumOfIntList(List<Double> list){
    // code
}

上面这两个方法共同点非常多，我们最好是把它们合并为一个方法，减少重复的代码。但问题是我们怎样合并呢，它们接收的方法参数是不一样的。

我们细心观察一下就会发现，List<Integer> 和 List<Double> 都是 List<E> 具体化后的类型。我们能不能提供一个具体化的类型，既能表示 List<Integer> 又能表示 List<Double> 呢。

这就是通配符的用武之地了。

注意，带通配符的类型不是泛型类型，而是泛型类型的具体化后的类型。只不过不是用具体某一个类型具体化的，而是用通配符具体化的。

# 上届通配符

语法：

<? extends A>

如：

List<? extends Number>

它可以表示 List<Number> ，也可以表示 List<任何 Number 的子类型> ，如：List<Integer> ， List<Double> 等等。

示例：

public static double sumOfList(List<? extends Number> list) {
    double s = 0.0;
    for (Number n : list)
        s += n.doubleValue();
    return s;
}

sumOfList 方法可以接收 List<Integer> ，List<Double> 或任何元素为 Number 的子类型的 List 。

调用：

List<Integer> li = Arrays.asList(1, 2, 3);
System.out.println("sum = " + sumOfList(li));

或

List<Double> ld = Arrays.asList(1.2, 2.3, 3.5);
System.out.println("sum = " + sumOfList(ld));

# 无界通配符

语法：

<?>

如：

List<?>

它可以表示 List<任意类型> 。

示例：

public static void printList(List<?> list) {
    for (Object elem: list)
        System.out.print(elem + " ");
    System.out.println();
}

printList 方法可以接收 List<Object> ，List<Integer> ，List<Box> 等等。

我们可能会错误的觉得上面的方法换成如下形式也可以达到相同的效果：

public static void printList(List<Object> list) {
    for (Object elem : list)
        System.out.println(elem + " ");
    System.out.println();
}

实际上，该方法只能接收 List<Object> 类型的参数。

# 下届通配符

语法：

<? super A>

如：

List<? supter Integer>

它可以表示 List<Integer> ，也可以表示 List<任何 Integer 的父类型>

# 通配符和 Bounded Type Parameter 的区别

其实区别已经非常明显了，通配符是具体化时用的，Bounded Type Parameter 是 generic class/interface 或 generic method 用的。

# 通配符的继承关系

虽然 Integer 是 Number 的子类，但是 List<Integer> 并不是 List<Number> 的子类。事实上，它们两者没有什么直接的关系。

但是 List<? extends Integer> 却是 List<? extends Number> 的子类，这是因为前者所能表示的范围被包含在后者所能表示的范围内。
