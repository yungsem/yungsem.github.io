# 七大原则

# 单一职责原则

一个类只负责一项职责，一个方法只干一件事情。

# 开闭原则（OCP原则）

对扩展（提供方）开放，对修改（使用方）关闭。

# 依赖倒置原则

依赖接口或抽象类，而不要依赖具体的实现类。也即面向接口编程。

依赖传递的方式：

  • 方法的形参
  • 成员变量 + 构造方法
  • 成员变量 + setter() 方法

# 迪米特法则

类对自己依赖的类知道的越少越好，也即最少知道。

要遵守迪米特法则，需要做到只与直接朋友通信，被依赖的类不要以局部变量的方式出现在类中。

直接朋友：

  • 成员变量
  • 方法参数
  • 方法返回值

# 里氏替换原则

在使用父类对象的地方，把父类对象替换成子类对象，不会改变程序的行为。

我们使用继承时，要想遵循里氏替换原则，就必须做到子类尽量不要重写父类的方法。因为子类一旦重写父类的方法，当把父类对象替换成子类对象（里氏替换）时，就会改变程序的行为，违背里氏替换原则。

# 合成复用原则

多用组合或聚合，少用继承。
