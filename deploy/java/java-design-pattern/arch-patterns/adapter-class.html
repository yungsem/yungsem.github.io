# 适配器模式：类适配器

在现实生活中，适配器的使用随处可见。比如你可能拥有一部手机，它的耳机接口是 Type-C 的，而你手头上只有一副 3.5mm 插头的耳机，现在你想用这副耳机听歌，该怎么办呢？显然，你会使用一个转接头。这个转接头就是一个适配器。

抽象一下，现在我们有一个接口 ，名为 Target ，定义如下：

public interface Target {
    void request();
}

正常情况下，如果我们要使用 Target 接口的 request() 方法，会先定义一个类实现 Target 接口，如下：

public class ConcreteTarget implements Target {
    @Override
    public void request() {
        System.out.println("=====>>>request from ConcreteTarget");
    }
}

然后像下面这样使用：

Target target = new ConcreteTarget();
target.request(); // output: =====>>>request from ConcreteTarget

这个过程就像你有一个 Type-C 接口的手机（Target 类），恰巧你又有一个 Type-C 插头的耳机（ConcreteTarget 类），无缝对接就可以听歌了（request() 方法）。

实际情况远没有这样巧。

假设你有一个类，名为 Adaptee ，它依据自己要提供的功能而定义，并没有实现 Target 接口，如下：

public class Adaptee {
    public void specificRequest() {
        System.out.println("=====>>>specificRequest from Adaptee");
    }
}

现在我们有一个需求：让 Adaptee 类的 specificRequest() 方法适配 Target 接口。

一种方法是改写 Adaptee 类的定义，直接实现 Target 接口，然后在实现 request() 方法时，间接调用 specificRequest() 方法，如下所示：

public class Adaptee implements Target {
    public void specificRequest() {
        System.out.println("=====>>>specificRequest from Adaptee");
    }

    @Override
    public void request() {
        this.specificRequest();
    }
}

使用：

Target target = new Adaptee();
target.request(); // =====>>>specificRequest from Adaptee

上述这种方法虽然比较直接，但是要修改现有的 Adaptee 类，如果 Adaptee 类参与了比较复杂的逻辑，修改成本非常高，这种方法就不是一个最佳选择。

此时，适配器就派上用场了。我们可以新定义一个类，名为 Adapter ，让 Adapter 类直接实现 Target 接口，同时继承 Adaptee 类，在实现 request() 方法时，调用继承而来的 specificRequest() 方法，如下所示：

public class Adapter extends Adaptee implements Target {
    @Override
    public void request() {
        super.specificRequest();
    }
}

使用：

Target target = new Adapter();
target.request(); // =====>>>specificRequest from Adaptee

借用适配器（Adapter），我们既实现了我们的原始需求，又没有改动我们的 Adaptee 类，扩展性比较强。
